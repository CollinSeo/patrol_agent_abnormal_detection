## 현재 결정 사항

### ADR-001. 비대화형 에이전트로 정의

- 날짜: 2026-04-09
- 상태: accepted
- 배경: 본 시스템은 현장 입력 이미지를 자동 처리하는 파이프라인 목적이 강하다.
- 결정: 사용자 채팅형이 아니라 입력 이미지 기반 자동 실행형 에이전트로 설계한다.
- 이유: 운영 흐름에 맞고 대량 케이스 처리 및 하네스 평가에 유리하다.
- 대안: 챗봇형 인터페이스
- 영향: 출력 스키마와 배치 실행 구조가 우선된다.

### ADR-002. 모델 추상화 계층 필요

- 날짜: 2026-04-09
- 상태: accepted
- 배경: 현재는 GPT-5.4로 개발하지만 향후 사내 모델 Kimi-K2.5로 전환 예정이다.
- 결정: 모델 호출부를 인터페이스로 분리한다.
- 이유: 모델 교체 비용을 줄이고 하네스 비교 실험을 쉽게 하기 위함이다.
- 대안: 모델 API 직접 의존 구조
- 영향: 설정, 프롬프트, 응답 파싱 계층을 분리해야 한다.

### ADR-003. 일반 이상 판정은 VLM 우선

- 날짜: 2026-04-09
- 상태: accepted
- 배경: 현장 이미지는 동일 위치 촬영이라도 미세한 앵글 차이와 조명 변화가 빈번하다.
- 결정: rule-based는 구조적 오류 탐지에 한정하고, 일반 케이스의 이상 여부 판단은 반드시 VLM을 거친다.
- 이유: 단순 이미지 비교만으로는 실제 이상과 환경 변화를 안정적으로 구분하기 어렵다.
- 대안: rule-based 1차 판정 후 선택적으로 VLM 수행
- 영향: VLM 호출이 기본 경로가 되며, 평가 하네스도 이를 전제로 설계한다.

### ADR-004. 정렬 실패는 앵글 변경으로 종료

- 날짜: 2026-04-09
- 상태: accepted
- 배경: 정렬 자체가 성립하지 않는 경우는 촬영 위치/각도가 잘못 수집된 사례일 가능성이 높다.
- 결정: 정렬 실패 시 후속 분석을 계속하지 않고 `앵글 변경` 유형으로 종료한다.
- 이유: 잘못된 입력을 정상 이상 분석 흐름에 태우면 오탐과 설명 오류가 커질 수 있다.
- 대안: 비정렬 상태로 계속 분석
- 영향: 전처리 단계에 종료 조건이 생기며, taxonomy에 `앵글 변경` 라벨이 필요하다.

## 미해결 이슈

- 조치 권고 수준과 표현 수위 기준 미확정
- 평가 데이터셋 정답 형식 미확정
- CLI 인자 구조 미확정
