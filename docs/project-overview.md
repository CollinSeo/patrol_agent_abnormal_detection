# 프로젝트 개요

## 목표

본 프로젝트의 목표는 공장/작업장 순찰 과정에서 수집되는 이미지 쌍을 비교 분석하여 이전 정상 상태 대비 현재 이미지의 변경 사항이 무엇인지 판단하고, 해당 변경이 이상에 해당하는 경우 유형을 분류하며, 그 결과를 바탕으로 후속 조치 권고까지 제시하는 VLM 기반 에이전트를 구축하는 것이다.

이 에이전트는 사용자의 채팅 요청에 응답하는 형태가 아니라, 지정된 입력 이미지 세트를 받아 자동으로 분석을 수행하는 비대화형 에이전트로 정의한다.

## 배경

- 현장 순찰 이미지에는 단순 배치 변화부터 실제 위험 상황까지 다양한 변경이 존재한다.
- 동일 위치, 동일 각도로 촬영하더라도 미세한 앵글 차이와 조명 변화가 지속적으로 발생한다.
- 이 때문에 픽셀 기반 직접 비교만으로는 실제 이상과 촬영 조건 변화를 안정적으로 구분하기 어렵다.
- 1차 이상 검출 알고리즘은 이상 여부와 후보 부위를 좁혀주는 역할을 수행한다.
- 최종적으로는 VLM이 시각 정보와 맥락을 해석하여 이상 유형과 조치 방향을 도출한다.

## 핵심 설계 원칙

- rule-based 비교는 극단적인 이미지 차이, 입력 누락, 명백한 앵글 오류 같은 구조적 실패를 걸러내는 용도로 제한한다.
- 일반적인 이상 변경 판정은 반드시 VLM 분석을 거쳐야 한다.
- 정렬 실패는 단순 분석 실패가 아니라 `앵글 변경` 케이스로 간주하고 해당 케이스 분석을 종료한다.

## 에이전트 역할

에이전트는 공장의 안전관리 담당자 관점에서 동작한다.

구체적으로는 반도체/제조 공장의 3S Patrol 전문 관리자 관점에서 이미지를 해석한다.

- 화재, 누수, 연기, 파손, 장애물 등 안전 리스크를 식별한다.
- 3S(정리, 정돈, 청결) 관점의 관리상 이상도 폭넓게 식별한다.
- 이상이 아닌 단순 변경과 실제 대응이 필요한 변경을 구분한다.
- 결과를 사람이 검토하기 쉬운 형태로 정리한다.

## 입력 정의

각 분석 단위는 하나의 하위 폴더로 관리되며, 기본적으로 다음 3종의 이미지로 구성된다.

1. 현재 수집된 원본 이미지
2. 이전에 수집된 정상 이미지
3. 두 이미지를 비교한 1차 이상 변경 시각화 이미지

1차 시각화 이미지 해석 규칙:

- 파란색 테두리 내부: 이상 변경 탐지 알고리즘이 중요하게 본 관심 영역
- 빨간색 패치: 실제 이상 변경이 발생했을 가능성이 높은 추정 영역

정렬 및 마스킹 관련 전제:

- 제공된 비교용 이미지는 이미 기하학적 정렬이 완료된 상태로 입력된다고 가정한다.
- 정렬 불가능한 가장자리 영역은 검은색으로 마스킹될 수 있다.
- 검은색 마스크 영역은 정보가 없는 영역으로 간주하고 분석 대상에서 제외한다.
- 카메라 앵글 차이는 대부분 보정되었으나, 미세한 정렬 오차는 남을 수 있다.

## 기대 산출물

에이전트는 최소한 아래 결과를 생성해야 한다.

1. 단어 중심의 간단한 이상 변경 요약
2. VLM 분석 과정에서 생성된 전체 텍스트 로그
3. 통합 정리된 최종 이상 검출 유형 결과
4. 이상 유형 기반의 조치 권고 사항

상기 항목은 최종 구조화 출력의 필수 구성 요소로 관리한다.

## 범위

### 포함 범위

- 이미지 로드 및 입력 구조 검증
- 이미지 정렬/전처리
- 이상 패치 영역 활용
- 전체 이미지 + 패치 기반 VLM 분석
- 결과 통합 및 최종 유형 도출
- 조치 권고 생성
- 평가 하네스 설계 및 검증 체계 수립

### 제외 범위 또는 추후 고도화 범위

- 사내 모델 실제 연동 전환
- 정교한 조치 매뉴얼 기반 운영 정책 적용
- 실시간 스트리밍 입력 처리
- 대화형 UI/챗 인터페이스

## 성공 기준 초안

- 입력 폴더 구조를 안정적으로 읽고 분석을 수행할 수 있어야 한다.
- 전체 이미지와 관심 패치를 함께 반영한 이상 분석 결과를 생성해야 한다.
- 최종 결과가 현장 담당자 관점에서 이해 가능한 수준의 유형명과 조치 권고를 제공해야 한다.
- 하네스 기반 평가에서 일관된 품질 점검이 가능해야 한다.
